## What Landed

### Part 1: Status bar size restoration ‚úÖ

Fixed sizes ‚Äî same in ALL states (10ft glanceability invariant):
- Clock: 68pt (`Instrument Serif`)
- All temps (outdoor, system, target): 60pt ‚Äî identical, matched to each other

No `isAmbient` size switching. The label bar text is small (`labelSm`) at 2ft viewing distance ‚Äî different jobs, different sizes.

### Part 2: Status bar architectural restructure ‚úÖ

**Final architecture** (iterated via iOS Simulator to get right):

- `HStack` with `Spacer(minLength: 20)` between groups ‚Üí fills full panel width
- Each number is in its own `VStack` paired with its label directly below:
  - System temp VStack ‚Üí `barLabel("SYSTEM")`
  - Target temp VStack ‚Üí `barLabel("TARGET")`
  - Front door VStack ‚Üí `barLabel("FRONT")`
  - Garage VStack ‚Üí `barLabel("GARAGE")`
  - Mic VStack ‚Üí `barLabel("VOICE")`
- Labels always rendered with `.opacity(showLabels ? 1 : 0)` ‚Äî NOT conditional `if showLabels`. This keeps column widths identical in ambient and engaged so Spacer distribution is invariant. Without this, adding labels changes icon column widths, shifts Spacers, and moves the left-side content.
- Arrow padded `.top` by `(tempSize - arrowHeight) / 2 = 16pt` to center it on the number midline (not top-align, which made it shoot over the numbers)
- Mic button gets outer `frame(60, 60)` to match `AlertGlowIndicator`'s `glowSize: 60` ‚Äî keeps all three icons at identical vertical position
- Labels shortened: TIME/OUTSIDE removed (self-evident). FRONT DOOR ‚Üí FRONT (fits cleanly)
- Weather conditions always show in both ambient and engaged ‚Äî not gated on `showLabels`
- Separate `StatusLabelBar` struct eliminated ‚Äî labels live inside each column

**Structural guarantee:** labels are under their elements because they are inside the same `VStack` as their element. No Spacer approximation, no pixel nudging, works on any screen width.

### Part 3: Shopping + Activity icons ‚úÖ

**Root cause:** `Spacer(minLength: 0)` in `QuickControlContent` VStack. The ZStack containing both `QuickControlContent` and `FullControlContent` (opacity-switched) is as tall as `FullControlContent`. The Spacer expanded to that height, pushing icons off-screen.

**Fix:** Removed `Spacer(minLength: 0)`. Icons render directly below dimmer with fixed spacing.

## Interaction Notes

**ZStack opacity-switching constraint:** Any `Spacer(minLength: 0)` in `QuickControlContent` will expand to `FullControlContent`'s height (even at opacity 0). Always use fixed spacing in the quick control body, never Spacer-to-bottom.

**Always-rendered labels:** The key insight for label alignment stability. `if showLabels` is tempting but wrong ‚Äî it changes column widths and shifts Spacers. `.opacity()` is the right tool here.

**Arrow vertical centering:** `HStack(alignment: .top)` aligns to tops of VStacks. Since VStacks include an invisible label below the number (adding ~19pt to height), the arrow's natural top matches the number's top ‚Äî but the arrow points upward, so it shoots above. Explicit top padding `(tempSize - arrowHeight) / 2` centers it correctly.
